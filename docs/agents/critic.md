# Critic Agent Specification (Reviewer)

## 职责
模拟人类审阅者，对幻灯片截图进行视觉与内容的双重评估。

## 继承
继承自 `BaseAgent`。

## 核心方法

### `review(self, image_paths: List[str]) -> List[Dict]`
*   **输入**: 每一页幻灯片的截图路径列表。
*   **输出**: 结构化的反馈列表 (JSON List)。
*   **逻辑**:
    1.  构造 Prompt，要求通过 Vision 识别图片中的问题。
    2.  **Prompt 重点**:
        *   **文字密度**: 是否字数过多，难以阅读？
        *   **排版布局**: 是否对齐？是否有遮挡？留白是否合适？
        *   **视觉层级**: 标题是否明显？重点是否突出？
        *   **一致性**: 字体、颜色风格是否统一？
    3.  **输出格式要求**:
        ```json
        [
          {
            "page_index": 1,
            "severity": "CRITICAL",  // CRITICAL, MAJOR, MINOR, INFO
            "issue": "Text overflow in code block",
            "position": "Bottom left",
            "suggestion": "Reduce font size or split code into two slides."
          }
        ]
        ```
    4.  调用 LLM (Vision Mode)。
    5.  解析 JSON 返回值。

## Prompt Design (System Prompt)
> You are a professional Presentation Critic. Your goal is to review slides generated by an automated system and provide constructive, actionable feedback to improve their visual quality and readability.
> You will receive images of slides. You must analyze them for:
> 1. Content Density: Is there too much text?
> 2. Layout: Are elements aligned? Is there overlapping?
> 3. Visuals: Are fonts readable? Is contrast sufficient?
>
> Return your feedback strictly in the specified JSON format. If a slide is perfect, return an empty list for that page or mark it as "PASS".

## 关键配置
*   需要使用支持 Vision 的模型 (`gpt-4o`).
*   可能需要分批处理图片（如果 PPT 页数过多），避免单次请求 Token 超限。
